# Shared Context

## Contents

- Consistent user definitions across all artifacts

### Glossary
- Project terminology and definitions
- Acronym reference
- Domain-specific language

### OKRs
- Shared objectives and key results
- Cross-functional goals
- Success metrics

### Style Guide
- Brand guidelines
- UI/UX standards
- Design system documentation

## Purpose

Maintain consistency and shared understanding across all project documentation and team members.

## How to Use

2. Consult personas for unified user understanding
3. Track OKRs collaboratively
4. Follow style guide for all deliverables
